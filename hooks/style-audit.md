# Stop hooks：文风审计 + manyread 重建

这里有两个挂在 `Stop` 事件上的 hook。第一个在每个回合结束前审计回复的文风并在不合格时打断改写，第二个在回合结束后重建 manyread 索引。两者都写进 `settings.json` 的 `hooks.Stop`，互不干扰。

## style-audit.mjs —— 文风审计（阻塞）

每次准备结束回合时，它取出本回合最终要展示给用户的回复文本，写一份临时文档到当前项目的 `archive/style-audit.md`，再起一个独立的、不带项目上下文的 `claude` 进程，按下面的平直语言标准给出裁决。裁决不通过时，hook 返回 `decision: block`，把回合打断，并把具体意见反馈给主 agent，要求改写后重交；通过则放行结束。

审计标准（必须全部满足）：

1. 平直：完整句子；不用电报体或碎片短语；不用箭头链（例如「A → B → 失败」）；不堆砌缩写；新造的代号或缩写在首次出现时给半句解释。
2. 非过度简化：简洁不等于潦草。为了短而丢掉读者据以行动所需的信息，算不通过。
3. 没有翻译腔，没有营销词。
4. 中文用全角标点；中文与英文或数字之间留一个空格。

内建的几条护栏：

- **隔离**：审计进程在临时目录里运行，只拿到「待审文本 + 上面的规则」，不读项目文件，避免被项目内容干扰判断。
- **防递归**：审计自己起的 `claude` 进程带环境变量 `CLAUDE_HOOK_NESTED=1`；hook 在开头看到这个变量就直接放行，避免嵌套审计无限派生。
- **防死循环**：同一条回复最多打断两次，仍不通过就放行并提示人工复核。
- **故障放行**：审计进程报错、超时或返回无法解析时，一律放行，绝不因为审计器本身坏掉而卡死会话。
- **省**：短于 280 字的回复（状态行、一句话汇报）跳过审计，不浪费模型调用。

## manyread-rebuild.mjs —— 重建索引（异步）

回合结束后异步运行，不阻塞回复。它先向上查找当前项目是否有 manyread store（`manyread/manyread.json`），没有就直接退出，不在任意仓库乱建索引。有 store 时，它判断源码自上次重建后是否有变化：git 仓库按提交与工作区指纹比对，无变化就跳过；非 git 仓库用 120 秒去抖。确实需要重建时，它先跑 `index_build.py` 重建 L1 索引，再跑 `enrich_treesitter.py` 补回 L2 的符号与边——因为 L1 的全量重建会清空符号表，必须紧接着富化，否则索引会退化成只剩全文检索。

插件脚本路径在运行时解析：优先取 `~/.claude/plugins/cache/manyread/manyread/<最新版本>/scripts/` 下的脚本，找不到再回退到 marketplace 检出，因此插件升级换版本后仍然可用。
